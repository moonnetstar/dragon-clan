### 1. 地圖相關類別結構（從 dex 真實解析）

APK 包名 `com.vidoar.motohud`（MotoEye v1.0.2604141705）。高德 SDK 佔 1557 個類型（`com.amap.api` 1397、`com.autonavi.*`），確認**完全沒有 Google Maps SDK**（`com.google.android.gms.maps` 類別數 = 0，僅有 material/errorprone 等非地圖庫）。原生庫含 `libAMapSDK_MAP_v10_1_302.so`（armeabi-v7a + arm64-v8a）。

App 自有地圖/導航類（共 51 個相關，關鍵如下）：

| 類別 | 角色 |
|---|---|
| `com.vidoar.motohud.view.NavigateActivity` | 主導航畫面，內嵌 `TextureMapView`/AMap 即時 3D 地圖、Marker、路線計算 |
| `com.vidoar.motohud.view.fragment.NavigateFragment` | 選點/開始導航，含 `startNavigateActivity(lat,lon,name)` |
| `com.vidoar.motohud.view.fragment.NaviViewFragment` / `NaviTypeFragment` / `NavigateSettingFragment` | 導航模式/設定 UI |
| `com.vidoar.motohud.view.CollectOnMapActivity` | POI 在地圖上選點收集 |
| `com.vidoar.motohud.view.GuideActivity` | 引導頁（含地圖教學） |
| `com.vidoar.motohud.utils.MapInfoUtils` | 地圖資訊工具 |
| `com.vidoar.motohud.bean.WayPoint` | 途經點資料 |

實際反編譯確認 `NavigateActivity` 使用 `AMap`、`MapView`、`BitmapDescriptorFactory`、`LatLng`、`Marker` 做即時渲染；`geo:0,0?q=` 字串已存在 dex（classes.dex）但非地圖主路徑，屬零散引用。

### 2. 兩種層次評估

**(A) 整個地圖渲染換 Google Maps SDK**
- 需：移除 AMap 依賴、改寫 `NavigateActivity`/`NavigateFragment` 全部地圖邏輯為 `com.google.android.gms.maps`、加入 Google Play Services、重打包 + 重簽、還要手機有 GMS。
- 問題：目標是安全帽 HUD 自用，實機多半為中國版/無 GMS 手機；且 AMap 有 1557 類 + native so，耦合極深。
- 工時：極高（改寫數千行 + 除錯），**不建議**。

**(B) 把「顯示地圖/導航」動作改成開啟 Google Maps 連結 Intent** ✅（推薦）
- 找到精準注入點：`NavigateFragment.startNavigateActivity(D D Ljava/lang/String;)V`
  - 它已經 `new Intent(context, NavigateActivity.class)` 並 `putExtra("lat",..)` / `"lon"` / `"addrName"` 後 `startActivity`。
  - 只要把這段改成 `Intent(Intent.ACTION_VIEW, Uri.parse("https://maps.google.com/?q=lat,lon"))` 再 `startActivity` 即可，繞過整個 AMap 渲染。
- 亦可對 `CollectOnMapActivity` 的選點結果做同樣處理。

---

### 3. 建議做法（優先 B 層次）
- 在 `startNavigateActivity` 處：攔截 lat/lon，改發 `ACTION_VIEW` 的 `geo:lat,lon?q=name` 或 `https://maps.google.com/?q=lat,lon`，交給手機上已裝的 Google Maps（或用 GMM 網頁）。
- 若想「顯示」而非「導航」，可改用 `https://www.google.com/maps/@lat,lon,15z` 開網頁版。
- 優點：不動 AMap native 庫、不需 GMS、工時最短、風險最低。

**預估工時（B 層次）：** 若只改 `startNavigateActivity` 一處 + 重打包重簽 ≈ **0.5–1 天**（含環境建置）；若要把多個選點入口都接上也約 **1–2 天**。

### 5. 給勞大的後續具體步驟
1. **本機只需這個 APK 就夠做靜態分析**（已證明），不需要先 root/adb pull 實機。
2. 若要動手改（B 層次）：
   - 裝 Java + jadx/apktool（或用 androguard 已可行）
   - `apktool d MotoEye.apk` 解出 smali → 改 `NavigateFragment.smali` 的 `startNavigateActivity` → `apktool b` → `apksigner` 重簽（需自製 keystore）
3. **實機部分**：
   - 若目標手機**有 Google Maps App + GMS**：直接 `geo:`/`maps.google.com` Intent 即可，不需 root。
   - 若手機**無 GMS**（如中國版 Android）：改用 `https://maps.google.com/?q=` 網頁 Intent 最穩，或裝 GMS 套件。
   - `adb pull` 實機資料只有在你想抓取「使用者收藏 POI / 帳號設定」等私有資料時才需要；純改地圖連結**不必 root**。
4. 注意：自簽 APK 安裝需允許「未知來源」；HUD 藍牙通訊（vidoar HUD）邏輯不受此改動影響，可並存。
