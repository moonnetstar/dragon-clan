## 三、修改內容明細

### （共通）拿掉登入機制
**檔案**：`smali_classes3/com/vidoar/motohud/view/SplashActivity.smali`

SplashActivity 啟動後由 `Handler`（SplashActivity$1）判斷 token：
- 有 token → `access$000` → `startMainPage()`（進主頁）
- 無 token → `access$100` → `startLoginPage()`（跳登入）

**改法**：把 `access$100` 內原本呼叫 `startLoginPage()` 改為呼叫 `startMainPage()`。
這樣**無論有沒有登入 token，永遠進主頁**，登入頁被徹底繞過。

```smali
.method static synthetic access$100(...)V
    invoke-direct {p0}, ...SplashActivity;->startMainPage()V   # 原本是 startLoginPage()
    return-void
.end method
```

> 選擇改 `access$100`（而非改 Handler 判斷）是最小侵入單點修改，兩條路徑都導向主頁，最穩。

---

### B 版：導航開 Google Maps 連結
**檔案**：`smali_classes3/com/vidoar/motohud/view/fragment/NavigateFragment.smali`
**方法**：`startNavigateActivity(DDLjava/lang/String;)V`（原 line 680–707）

**原行為**：`new Intent(context, NavigateActivity.class)` 帶 lat/lon/addrName → 跳高德地圖頁。

**改後行為**：組出字串 `https://maps.google.com/?q=<lat>,<lon>`，用
`new Intent(ACTION_VIEW, Uri.parse(url))` + `startActivity` 開啟。
手機有 GMS + Google Maps App 時會直接跳 Google Maps App；否則走瀏覽器網頁版。

```smali
# 組 "https://maps.google.com/?q=" + lat + "," + lon
const-string v1, "https://maps.google.com/?q="
... StringBuilder append(D) / append(",") / append(D) ...
new-instance v1, Landroid/content/Intent;
const-string v2, "android.intent.action.VIEW"
invoke-static {v0}, Landroid/net/Uri;->parse(...)
invoke-direct {v1, v2, v0}, Landroid/content/Intent;-><init>(String;Uri;)V
invoke-virtual {p0, v1}, ...->startActivity(Intent;)V
return-void
```

高德 SDK 完全保留，只改這一處行為。

### A 版：App 內地圖換成 Google Maps
**檔案**：`smali_classes3/com/vidoar/motohud/view/NavigateActivity.smali`
**方法**：`getContentView()`（原 line 911）

**做法（覆蓋法，最穩）**：保留原本高德 `ActivityNavigateBinding` 版面與生命週期不動，
在 `getRoot()` 回傳的 `RelativeLayout` 上，**動態 addView 一個全螢幕 WebView**，
載入 Google Maps 嵌入頁：

```
https://maps.google.com/maps?q=<lat>,<lon>&z=15&output=embed
```

lat/lon 由 `getIntent().getDoubleExtra("lat"/"lon", 0)` 取得（與原 NavigateActivity 讀取 Intent 的邏輯一致，NavigateFragment 仍照原樣帶 extra 進來）。
WebView 開啟 JavaScript、`MATCH_PARENT` 全螢幕覆蓋在高德 MapView 之上，
使用者看到的即是 **Google 地圖畫面**。

```smali
new-instance v1, Landroid/webkit/WebView;
invoke-direct {v1, p0}, ...WebView;-><init>(Context;)V
... getSettings().setJavaScriptEnabled(true) ...
# 組 https://maps.google.com/maps?q=lat,lon&z=15&output=embed
invoke-virtual {v1, v2}, ...WebView;->loadUrl(String;)V
new-instance v2, Landroid/widget/RelativeLayout$LayoutParams; (-1,-1)
invoke-virtual {v0, v1, v2}, RelativeLayout;->addView(View;LayoutParams;)V
return-object v0
```

> 為何用覆蓋而非砍掉高德 MapView：NavigateActivity 大量 smali（marker、路線、生命週期 onCreate/onResume/onPause 都綁 mvRoute）依賴 binding，若直接抽掉 MapView 會全面 NPE。**覆蓋法零風險、改一個方法即可**，App 內顯示即為 Google 地圖。

## 六、已知限制 / 風險

1. **簽章不同**：自簽版無法覆蓋原廠版，需先 uninstall；且原廠雲端若做簽章校驗，登入/同步類 API 可能拒絕。
2. **登入拿掉後可能失效的功能**：
   - 帳號綁定的 **POI 收藏 / 路線雲端同步**（token 為空，後端會 401）。
   - 個人資料、訂單、韌體 OTA（若需登入態）可能無法使用。
   - **不受影響**：地圖顯示、導航跳轉、藍牙 HUD 連線、本地設定 — 這些不依賴登入 token。
3. **A 版 WebView 需連網**：Google Maps embed 需要網路 + GMS 環境；離線時只會顯示空白/載入中。已確認 App 具 `INTERNET` 權限。
4. **A 版底層高德仍在跑**：高德 MapView 仍在 WebView 下方初始化（不可見），略耗資源但不影響顯示；換來的是零 NPE 風險。
5. **座標系差異**：高德回傳的是 GCJ-02（火星座標）。中國大陸範圍內 Google Maps 用 GCJ-02 的 `q=` 參數大致對得上；若在境外或需高精度，座標可能有數十~百公尺偏移。此為座標系本質差異，非改機 bug。
6. **B 版 `?q=lat,lon`**：只帶終點座標，不帶起點/路線規劃；如需「規劃路線」可改用 `https://www.google.com/maps/dir/?api=1&destination=lat,lon`（本次採最穩的 `?q=` 定位版）。
