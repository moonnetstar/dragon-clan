# 專屬代駕助理 App — 技術研究報告
# 工程師布魯斯 撰寫
# 日期：2026-06-03

## 願景
將龍族 Agent 放入汽車，成為「專屬代駕助理」
未來整合特斯拉或其他車廠的自動駕駛系統

---

## 一、技術架構

### 車載系統分層
```
┌─────────────────────────────────┐
│     龍族 Agent 層（決策）        │
│  語音互動 + AI 推理 + 任務規劃   │
├─────────────────────────────────┤
│     中介軟體層（通訊）           │
│  MQTT / gRPC / WebSocket        │
├─────────────────────────────────┤
│     車載 OS 層（控制）           │
│  Android Automotive / QNX / Linux│
├─────────────────────────────────┤
│     硬體層（感測器 + 執行器）    │
│  相機 / 雷達 / 超音波 / CAN Bus  │
└─────────────────────────────────┘
```

### 龍族 Agent 在車上的角色
1. **語音助理** — 語音控制導航、空調、音樂
2. **導航規劃** — 最佳路線、即時路況、POI 推薦
3. **駕駛監控** — 疲勞偵測、注意力提醒
4. **車輛管理** — 油量/電量、保養提醒、故障診斷
5. **安全預警** — 行人偵測、碰撞預警、天氣提醒

## 二、自動駕駛系統整合方案

### 方案 A：Tesla FSD（全自動駕駛）
- **優勢**：最成熟的 L4 自動駕駛，OTA 持續更新
- **挑戰**：Tesla 不開放第三方 API，只能透過手機 App 控制
- **龍族角色**：語音助理 + 導航規劃 + 行程管理
- **整合方式**：Tesla App API（有限）+ 藍牙/WiFi 語音

### 方案 B：Android Automotive OS
- **優勢**：Google 開放平台，可安裝第三方 App
- **支援車廠**：Volvo、Polestar、GM、Ford、Honda
- **龍族角色**：完整 Agent 體驗
- **整合方式**：原生 Android App + Google Assistant SDK

### 方案 C：CarPlay / Android Auto
- **優勢**：幾乎所有新車都支援
- **挑戰**：功能受限，只能顯示通知和簡易互動
- **龍族角色**：通知推播 + 語音助理（Siri/Google）
- **整合方式**：CarPlay Extension / Android Auto App

### 方案 D：獨立車機（Aftermarket）
- **優勢**：不限車型，可安裝在任何車
- **產品**：Android 車機（如 Pioneer、Sony）
- **龍族角色**：完整 Agent 體驗
- **整合方式**：原生 Android App

## 三、推薦方案（保守穩健）

### 短期（1-6 個月）：CarPlay + 手機 App
- 開發 iOS App 支援 CarPlay
- 語音助理用 Siri Shortcuts
- 導航用 Apple Maps / Google Maps API
- 成本低、風險小、所有車都能用

### 中期（6-18 個月）：Android Automotive App
- 開發 Android Automotive 版本
- 整合 Google Assistant
- 支援 Volvo/Polestar 等車廠
- 完整語音 + 導航 + 車輛管理

### 長期（18 個月+）：深度整合
- 與車廠合作開發原生車載 Agent
- 整合 CAN Bus 讀取車輛數據
- OTA 更新 Agent 能力
- 邁向 L4 自動駕駛協作

## 四、核心功能規劃

### MVP（最小可行產品）
1. 語音導航（「帶我去最近的加油站」）
2. 路況提醒（「前方 3 公里塞車，建議走替代路線」）
3. 停車提醒（「你的車停在 B2-158 號」）
4. 油耗/電量管理（「目前電量 30%，建議在下一个充電站充電」）
5. 行程規劃（「明天 8 點要到機場，建議 6:30 出發」）

### 進階功能
1. 疲勞駕駛偵測（手機相機）
2. 語音購物（「幫我買一杯星巴克」→ 導航到最近的）
3. 智慧停車（自動找車位、導航到空位）
4. 車輛共享（家人共用車輛、權限管理）
5. 保險整合（UBI 車險、駕駛行為評分）

## 五、技術挑戰

| 挑戰 | 解決方案 |
|------|---------|
| 車載系統碎片化 | 先用 CarPlay/AA 覆蓋最大範圍 |
| 語音辨識車內噪音 | 用車用麥克風陣列 + 降噪演算法 |
| 網路連線不穩定 | 離線語音 + 快取機制 |
| 安全關鍵系統 | 不直接控制車輛，只提供建議 |
| 法規限制 | 遵循各地區自動駕駛法規 |
