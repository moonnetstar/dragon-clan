## 今日學習重點：向量化回測 vs. 事件驅動回測

在策略回測中，最常見的兩個架構是 **向量化回測（Vectorized Backtesting）** 和 **事件驅動回測（Event-Driven Backtesting）**。理解兩者的差異，能讓龍族成員在實作時做出正確的架構選擇。

### 1. 向量化的核心思維

向量化回測的最大優勢在於「把迴圈丟給 NumPy」：只要回測邏輯是全歷史的函數型操作（函數輸入是整條價格序列，輸出也是對應長度的訊號序列），就能用 Pandas / NumPy 一次算完。Python 的 for-loop 在大量交易筆數下非常慢（每支股票 × 每根 K 棒都要跑 Python 層級的判斷），但向量化操作走的是 C/Fortran 層級，速度快上數量級。

### 2. EMA-9/21 向量化策略（完整可執行）

```python
"""
EMA-9 / EMA-21 多單策略 — 向量化回測
用途：輸入 close 序列 → 輸出帶 position + trade_pnl 的 DataFrame
執行：pip install pandas numpy; python3 ema_vectorized_backtest.py
Author: 龍族 AI 技術筆記 by Dorothy & Lao Da (2026-07-08)
"""

import numpy as np
import pandas as pd


def vectors_ma_strategy(
    closes: pd.Series, fast_period: int = 9, slow_period: int = 21
) -> pd.DataFrame:
    """輸入收盤價序列，輸出完整交易日誌。"""
    df = pd.DataFrame(index=closes.index, columns=["Fast", "Slow"])
    df["Fast"] = closes.ewm(span=fast_period).mean()
    df["Slow"] = closes.ewm(span=slow_period).mean()

    # 訊號：快線在慢線上 ⇒ +1 (持倉)；反之 −1 (空)
    signal      = np.sign(df["Fast"].values - df["Slow"].values)
    position    = pd.Series(
        np.nan_to_num(signal, nan=0), index=closes.index
    ).shift(1).fillna(0).astype(int)   # 隔夜持有 (avoid look-ahead)

    trade_entry_time, trades_df = _build_trades(position)

    return pd.DataFrame({
        "close":            closes.values,
        "fast_ma":          df["Fast"].values,
        "slow_ma":          df["Slow"].values,
        "position":         position.values.astype(int),
        "trade_entry_time": trade_entry_time,     # 每筆成交的 Entry Index
    }, index=closes.index)


def _build_trades(position: pd.Series):
    """從持倉序列中提取「每一段連續持倉」的起始時間索引。"""
    entries = []
    i_enter = None
    for i in position.index:
        pos     = int(position.loc[i])
        was_held = (i_enter is not None)

        if  pos == 1 and not was_held:              # ← Open Long
            i_enter = i
        elif pos != 1 and was_held:                  # → Close Position
            entries.append(i_enter)
            i_enter = None
    return entries, pd.DataFrame({"entry": entries})


def stats(prices: pd.Series, position: pd.Series):
    """計算回測績效：年化報酬率、夏普比。"""
    rets   = prices.pct_change() * position.fillna(0)      # 策略收益序列
    cum    = (1 + rets).cumprod() - 1                      # 累積淨值曲線
    return {
        "total_return": float(cum.iloc[-1]),
        "annualised":   float((cum.iloc[-1] + 1) ** (252 / len(cum)) - 1),
        "sharpe_ratio": float(rets.dropna().mean() / rets.std()) * np.sqrt(252),
    }


# ────────────── Mock-Price 回測範例 ──────────────

if __name__ == "__main__":
    rng = np.random.RandomState(42)
    n   = 250  # ~1 trading year (日線)
    closes = pd.Series(
        data=100 * np.exp(rng.normal(0.03, 0.25, size=n).cumsum()),
        name="Close",
    )

    df = vectors_ma_strategy(closes)
    result = stats(df["close"], df["position"])
    
    print("=" * 60)
    print("🔧 EMA-9/21 Vectorized Backtest")
    print("=" * 60)
    for k, v in result.items():
        print(f"  → {k.replace('_', ' ').capitalize()}: {v:.4%}")

    # === Trade P&L using vectorized np.diff ===
    positions = df["position"].fillna(0).values
    pnl_vec   = pd.Series(np.diff(positions, prepend=0), index=df.index)

    # 平倉損益 (exit → entry close shift(-1)) : -1 × Δclose
    exit_pnl  = (-pnl_vec[pnl_vec < 0] * df["close"].diff().shift(1)).dropna()
    
    n_trades  = int(abs(pnl_vec).sum())  # +開倉, −平倉
    print(f"  → Total Trades:        {n_trades}")
    print(f"  → Close Positions:     {len(exit_pnl)}")
```

### 3. 完整交易紀錄（每筆交易狀態與進出價格）

回測的核心，是追蹤每一下單時間點所對應的交易進場出場資訊。以上述的回測資料為基礎：
- `position` 從 0 跳至 1，即代表持有多張的時點；同樣地，由 1 退回 0 代表平倉的時間。

使用 `np.diff()` 找到這些斷裂點即可算出完整的進出場記錄與每當期損益：
- Entry / Exit price: K 條上的即時價  
- Hold Time（天）/ PnL%：`(Exit Price − Entry Price) ÷ Entry Price`  

向量化計算範例：

```python
import numpy as np

# position_diff = +1 (開倉), -1 (平倉), 0 (續持)
position_changes = np.diff(position.fillna(0).values, prepend=0)

# 計算每段持倉的損益比率（Exit Price / Entry Price − 1）
df["trade_pnl"] = position_changes * df["close"].pct_change().shift(-1)
```

- **Walk-forward validation：** 每次只看近 N 個月的訓練集。測試期往前推進。
