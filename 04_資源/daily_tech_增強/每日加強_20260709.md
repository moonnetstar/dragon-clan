## 🎯 今日學習點：Tailwind CSS + Framer Motion 元件互動設計

### 🔑 核心概念：Why Tailwind + Framer Motion?
現代 React/Next.js 前端團隊普遍採用 **Tailwind CSS**（實用優先工具類框架）搭配 **Framer Motion**（React 生態系最強動畫庫）。  
- **Tailwind → UI 佈局與樣式處理**。  
- **Framer Motion → 觸發回饋、頁面流轉、元件級連動。**  

這組技術棧的好處：**設定檔最小化、可維護性高，且支援 RWD**。  

### 💡 今日實戰重點：`framer-motion`「Hover / Tap」回饋設計
元件回饋（Feedback）是 UX 最基礎也最重要的環節。  
以 Framer Motion 打造 Hover 動畫：

```jsx
<BoxAnimate
  whileHover={{ scale: 1.05 }}
  whileTap={{ scale: 0.98 }}
>
  <Button>點擊我</Button>
</BoxAnimate>
```

Framer Motion 自動處理觸控裝置的 `tap`，而不需要額外寫 `useEffect(...)`。  

### 🛠️ 實際應用情境：卡片列表 Hover 效果
**情境說明：**  
產品/數據卡片需要「Hover 時浮起」+ 「點擊時微縮」。這比 hover CSS 更好看的原因是 Framer Motion 支援 **`layoutId` 連動**。

```jsx
import { motion } from "framer-motion";

function Card({ children }) {
  return (
    <motion.div 
      layout
      transition={{ type: "spring", stiffness: 300, damping: 25 }}
      whileHover={{ y: -8, boxShadow: "0px 10px 24px rgba(0,0,0,0.2)" }}
    >
      {children}
    </motion.div>
  );
}
```

### 📌 使用時機建議：
- ✅ List / Card 浮動效果  
- ✅ Sidebar 拉入拉出 (Layout Transition)  
- ❌ 不適合用於「頁面轉場」(除非 SPA singlePage)，這時改用 Next.js built-in `<Link>` anim
